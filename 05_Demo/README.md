# 05 — Demo

Everything needed to walk a judge through WikiGenius in 3–5 minutes.

> **Ready:** [`demo_script.md`](demo_script.md) — ~4-min beat-by-beat golden path · [`seed_data.md`](seed_data.md) — preload fixtures · [`dashboard/index.html`](dashboard/index.html) — HTML dashboard skeleton (Signal template; one screen per beat, ← / → to navigate; `open` it or serve `05_Demo/dashboard/`). Built on the confirmed B1 scenario (`../03_Design/b1_demo_scenario.md`).

## Suggested contents

- `demo_script.md` — beat-by-beat walkthrough with timing and talking points
- `talking_points.md` — the 3–5 things judges must remember after the demo
- `recordings/` — screen captures, backup video in case the live demo dies
- `qa_prep.md` — anticipated judge questions and prepared answers (especially "why two agents instead of one?")

## Demo discipline

- One golden path. Every other interaction is a distraction.
- The hard problem (multi-agent coordination) needs to be *shown*, not just claimed — script at least one moment where a conflict gets resolved on screen.
- Have a backup video. Live demos break.
