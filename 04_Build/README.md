# 04 — Build

Source code, prototypes, and technical experiments.

## Suggested contents

- `prototype/` — working code, however rough
- `experiments/` — throwaway spikes, notebooks, model probes
- `dev_notes.md` — running log of what worked, what didn't, decisions made mid-build
- `setup.md` — how to run the prototype locally (deps, env vars, commands)

## Working principles for the build phase

- Demoable beats complete — every commit should make the demo path more solid, not broader
- Cut scope ruthlessly when behind schedule; the proposal already lists what's *not* in MVP territory
- Log surprises immediately in `dev_notes.md` so they don't get lost
